# 需求画布工具

需求画布工具是一款面向产品经理、设计师、交互设计师和研发团队的本地 Web 工作台。它把需求信息、页面原型、页面流转、文档生成和 AI 上下文整理放在同一个界面里，帮助团队在需求早期快速建立一个可视化、可编辑、可交付的产品方案。

项目当前包含两种运行形态：

- 纯前端模式：没有后端时会自动使用浏览器本地存储，适合单机推演和轻量交付。
- 本地服务器模式：`server/index.mjs` 提供账号、设计单、上传文件、Markdown 导出和权限控制，数据写入 SQLite，图片/HTML/导出包写入本地或 NAS 文件目录。

项目还包含 `mcp/canvas-mcp-server.mjs`，可接入 Cursor、Claude、Codex 等 MCP 客户端，把聊天内容、HTML 原型和关键决策生成 Canvas 设计单与 PRD Markdown。

## 产品定位

传统需求梳理通常分散在文档、流程图、原型图、聊天记录和 AI 提示词中。这个工具的核心目标是把这些分散信息聚合为一份“可视化需求资产”：

- 在画布上看清页面关系。
- 在节点里沉淀页面原型和体验目标。
- 在分组中表达业务模块和流程区域。
- 在文档视图中自动生成结构化 PRD。
- 在 AI 上下文视图中复制给大模型继续扩写、检查或改写。
- 在交付入口中导出结构化数据，方便备份和传递。

## 适用场景

- 新功能需求从 0 到 1 的页面流程梳理。
- App 或 Web 产品改版时的页面节点整理。
- 把散落的原型截图、跳转逻辑和体验目标汇总成 PRD 初稿。
- 和设计、研发一起评审页面流、分支逻辑和异常路径。
- 给 AI 提供结构化上下文，生成更完整的需求文档或研发说明。
- 在本地快速制作一个可视化需求交付包。

## 当前版本概览

- 应用类型：本地单页 Web 应用 + 可选本地后端 + Canvas MCP Server
- 技术栈：React 19 + Vite 8

- 本地多用户服务：`npm run dev:local`，默认 `http://127.0.0.1:5180/canvas.html`

- 交付形态：源码 + README + 可选生产构建产物

## 核心功能

### 1. 需求信息

顶部“需求信息”用于维护需求的基础背景：

- 需求名称
- 需求背景
- 数据目标
- 整体体验目标
- 需求分析文档链接

这些信息会同步出现在文档视图和 AI 上下文中，作为整份需求材料的基础。

### 2. 流程画布

画布是产品方案的主要工作区。你可以在画布中创建页面节点、上传原型图、连接页面跳转、整理分支路径和调整整体布局。

支持的画布能力包括：

- 双击空白区域创建页面节点。
- 拖动节点调整位置。
- 双击节点打开页面详情侧边栏。
- 双击节点原型区域上传页面原型图。
- 从节点左侧锚点拖出连接上游页面。
- 从节点右侧锚点拖出连接下游页面。
- 点击连接线标签编辑跳转逻辑。
- 选中节点后右键复制、删除、成组或分享流程图。
- 按住空格拖动画布。
- 使用底部工具条缩放、归位和一键整理。
- 使用 `Cmd/Ctrl + Z` 撤销最近一次普通编辑。

### 3. 页面节点

页面节点采用卡片式视觉语言。节点结构为：

1. 页面标题
2. 原型图展示区
3. 页面体验目标说明

设计细节：

- 标题最多展示两行。
- 小字说明最多展示三行。
- 节点小字说明展示“页面体验目标”，不是页面说明。
- 没有体验目标时不展示占位提示。
- 上传原型图后，展示区按图片比例适配高度。
- 原型图展示区采用平滑圆角裁切。
- 未上传原型图时，展示区为轻量占位状态。
- 没有说明文字时，图片区底部到节点底部保留 12px 间距。

### 4. 页面详情侧边栏

双击节点会打开页面详情侧边栏。侧边栏从右向左滑出，动画时长约 0.3 秒，使用缓入缓出效果。

详情中可编辑：

- 页面名称
- 页面原型图
- 页面说明
- 页面体验目标
- 竞品参考图
- 竞品参考说明

页面详情是节点信息的主要录入入口，文档视图和 AI 上下文会复用这些数据。

### 5. 跳转逻辑

节点之间通过连接线表达页面跳转。连接线支持：

- 普通前向连接。
- 反向连接。
- 分支连接。
- 选中连接线后编辑跳转条件。
- 连接线中点显示跳转逻辑输入框。

连接线绘制会尽量避免重叠遮挡。反向连接会使用更大的曲线空间，并按路径冲突自动错开。

### 6. 分组

分组用于表达业务模块、页面集合或流程区域。它不是简单的视觉框，而是参与文档排序和流程组织的结构。

支持能力：

- 多选节点后成组。
- 拖动分组底板整体移动组内节点。
- 未分组节点拖入分组区域。
- 已成组节点可从右键菜单移出组。
- 分组可解组。
- 分组名称可编辑。
- 分组支持 10 种预设色板。
- 色板按钮位于组名称前方。
- 分组底板使用带透明度的颜色，避免和画布背景混在一起。

### 7. 文档视图

文档视图会基于画布数据生成一份结构化 PRD。它不是静态预览，而是可编辑文档。

文档包含：

- 一、需求背景
- 二、目标
- 三、总流程
- 四、页面明细

文档视图特性：

- 中央白色文档卡片承载正文。
- 左侧提供富文本格式工具栏。
- 右侧提供目录导航。
- 总流程预览图点击后自动回到画布视图。
- 页面明细支持按流程顺序或手动顺序展示。
- 页面明细标题右侧提供“排序”入口。
- 文档里的流程预览图不展示节点标题和小字说明，也不展示点阵背景。

### 8. 文档工具栏

文档视图左侧工具栏支持常用编辑功能：

- 加粗
- 斜体
- 下划线
- 删除线
- 一级标题
- 二级标题
- 三级标题
- 无序列表
- 链接
- 插入图片

这些工具作用于当前富文本编辑区，适合对需求背景、目标和页面描述做轻量排版。

### 9. 文档目录

文档右侧目录用于快速定位内容。

目录包括：

- 需求背景
- 目标
- 总流程
- 页面明细
- 分组标题
- 组内页面

点击目录项会滚动到对应章节。目录底部显示页面数量、文字数量和最后编辑日期。

### 10. 页面排序

文档页面顺序支持两种模式。

默认流程排序：

- 根据画布连线推导页面顺序。
- 优先从无入边页面开始。
- 分支内参考画布位置排序。
- 分组会作为一个流程块处理。
- 组内页面会尽量挨在一起展示。

手动排序：

- 点击页面明细标题右侧“排序”按钮打开排序弹窗。
- 可选择是否按组展示。
- 平铺模式下可拖拽全部页面。
- 按组展示时只允许组内拖拽。
- 保存后手动顺序会成为新的文档展示顺序。
- 可恢复默认流程排序。

### 11. AI 上下文

AI 上下文视图会把当前需求整理成 Markdown 文本，方便复制给大模型继续处理。

内容包括：

- 需求背景
- 数据目标
- 整体体验目标
- 页面列表
- 页面说明
- 页面体验目标
- 原型图是否存在
- 竞品参考
- 页面跳转关系
- 分支点统计

典型用途：

- 生成 PRD 初稿。
- 检查需求遗漏。
- 生成研发说明。
- 生成测试用例。
- 改写成汇报材料。

### 12. 交付链接

顶部“交付链接”按钮会复制当前设计单的结构化 JSON。为了保持轻量，交付 JSON 默认会把大体积图片字段替换成占位描述。

适合用于：

- 备份当前需求结构。
- 交给其他工具消费。
- 在团队内传递页面节点、跳转和分组关系。

## 视觉语言

当前界面以浅色玻璃画布为主：

- 背景使用浅灰蓝点阵网格。
- 节点使用白色卡片、轻边框和柔和阴影。
- 连接点使用左右输入/输出锚点。
- 连接线使用虚线和柔和蓝灰色，选中态增强蓝色。
- 分组底板使用透明色块和虚线边框。
- 文档视图使用白色页面卡片、左工具栏和右目录。
- 整体控件圆角克制，强调工作台感而不是营销页感。

视觉目标是让页面关系、节点状态和文档内容都能快速扫描，同时保留足够的原型图展示空间。

## 数据结构

核心数据对象名为 `doc`。

```js
{
  schema: "prd-canvas/1.0",
  meta: {
    name: "",
    background: "",
    dataGoals: "",
    expGoals: "",
    analysisUrl: "",
    date: "YYYY-MM-DD",
    setupDone: true,
    docOrder: [],
    docSortMode: "flow",
    docGroupView: true
  },
  nodes: [
    {
      id: "",
      x: 0,
      y: 0,
      name: "",
      note: "",
      expGoal: "",
      proto: null,
      protoRatio: 0.55,
      competitors: []
    }
  ],
  edges: [
    {
      id: "",
      from: "",
      to: "",
      label: ""
    }
  ],
  groups: [
    {
      id: "",
      name: "",
      nodeIds: [],
      colorIdx: 0
    }
  ]
}
```

字段说明：

- `meta`：需求信息、文档排序模式和文档展示偏好。
- `nodes`：页面节点，包括坐标、名称、说明、体验目标、原型图和竞品图。
- `edges`：页面跳转关系。
- `groups`：页面分组。
- `note`：页面说明字段。保留这个名称是为了兼容旧数据。
- `expGoal`：页面体验目标，是节点卡片底部小字说明的来源。
- `protoRatio`：上传原型图的高宽比，用于按图片比例适配节点展示区。

## 本地存储

数据存储键：

```text
prdcanvas:v1
```

存储位置：

```text
浏览器 localStorage
```

注意事项：

- 同一浏览器、同一域名和端口下会自动恢复上次编辑内容。
- 更换浏览器、清理站点数据或更换访问域名后，数据不会自动迁移。
- 上传图片会以 base64 存入本地数据，图片过多可能触发浏览器存储上限。
- 如果要长期保存，建议复制交付 JSON 或导出当前项目包。

## 快速开始

安装依赖：

```bash
npm install
```

启动本地开发服务：

```bash
npm run dev
```

默认访问：

```text
http://127.0.0.1:5173/
```

如果 5173 被占用，Vite 会自动使用下一个端口，例如：


构建生产版本：

```bash
npm run build
```

预览生产构建：

```bash
npm run preview
```

## 打包与交付

建议交付包包含：

- `README.md`
- `index.html`
- `package.json`
- `package-lock.json`
- `prd-canvas (2).jsx`
- `src/`
- `dist/`

建议排除：

- `node_modules/`
- `.DS_Store`
- `._*`
- 临时截图
- 日志文件
- 旧压缩包

原因：

- `node_modules/` 体积很大，可通过 `npm install` 重新生成。
- `dist/` 已包含当前生产构建产物，可用于静态部署。
- README 已包含产品说明和运行说明，适合随包交付。

## 部署方式

构建完成后，`dist/` 可以部署到任意静态站点服务：

- Nginx
- Vercel
- Netlify
- GitHub Pages
- 内网静态文件服务
- 本地 `npm run preview`

因为应用使用浏览器本地存储，不依赖服务端 API，所以静态托管即可运行。

## 维护指南

主要组件：

- `PRDCanvas`：应用壳、模式切换、数据加载与保存。
- `Canvas`：画布交互、节点、连线、分组、多选、复制粘贴。
- `NodeDetail`：页面详情侧边栏。
- `DocView`：文档视图。
- `DocFormatToolbar`：文档左侧格式工具栏。
- `DocToc`：文档右侧目录。
- `DocSortModal`：页面明细排序弹窗。
- `FlowThumb`：文档中的总流程缩略图。
- `AIViewPane`：AI 上下文生成与复制。
- `DeliverBtn`：交付 JSON 复制。

关键排序函数：

- `flowOrderItems`：根据节点和连线推导流程顺序。
- `docFlowBlocks`：把组和未分组节点整理成文档排序块。
- `buildDocPresentation`：根据排序模式和按组设置生成文档展示结构。
- `normalizeDocOrder`：清理手动排序，保证新增页面不会丢失。
- `buildSortDraft`：生成排序弹窗中的可拖拽草稿。

关键节点尺寸逻辑：

- `nodeTitleH`：计算标题区域高度。
- `nodeDescH`：计算体验目标说明区域高度。
- `defaultImgH`：计算原型图展示区默认高度。
- `routeNodeH`：计算节点整体高度。
- `routeAnchorY`：计算连接锚点垂直位置。

## 已知限制

- 当前没有账号系统。
- 当前没有后端存储。
- 当前没有多人实时协作。
- 当前没有正式测试套件。
- 上传图片会增加浏览器本地存储体积。
- 清理浏览器站点数据会清空当前需求。
- 交付 JSON 默认不携带原始图片大字段。
- 当前项目文件名 `prd-canvas (2).jsx` 带空格和括号，命令行引用时需要加引号。
